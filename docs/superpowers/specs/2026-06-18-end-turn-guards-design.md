# Защита кнопки «Завершить ход» от случайных нажатий

Дата: 2026-06-18

## Проблема

В игре с одного устройства (hot-seat) капитан завершает фазу подсказки и передаёт
телефон игрокам. В момент передачи (или просто случайно) кто-то может нажать
«Завершить ход» — ход обрывается, не сделав ни одной догадки.

## Решение

Два независимых UI-гарда на кнопке «Завершить ход» ([Board.tsx](../../../src/ui/screens/Board.tsx)).
Каждый из них действует во всех режимах (local / host / guest). Транспорт и редьюсер
не затрагиваются — это чистые гарды на стороне отображения.

### Якорь времени

Обе механики опираются на `turnStartedAt(view)` ([turnTimer.ts](../../../src/ui/turnTimer.ts)) —
метку выдачи активной подсказки из лога партии. В hot-seat это ровно момент передачи
телефона: после `giveClue` роль сразу переключается на `operative`, и тогда же логируется
подсказка. Якорь переживает перемонтирование экрана и синхронизируется гостям (на нём же
построен таймер хода).

### Гард 1 — блокировка с отсчётом (5 секунд)

В первые 5 секунд хода кнопка недоступна и показывает обратный отсчёт
`Завершить ход · 5 → 4 → … → 1`, затем активируется. Отсчёт идёт от `turnStartedAt`,
поэтому возврат из настроек его не перезапускает, а слишком долгая передача телефона
(> 5 c) застанет кнопку уже разблокированной — на этот случай работает гард 2.

- Чистая функция `endTurnLockLeft(startedAt, lockSec, now)` в `turnTimer.ts` — сколько
  секунд блокировки осталось (по образцу `turnSecondsLeft`). Покрыта юнит-тестом.
- Хук `useEndTurnLock(view)` в [hooks.ts](../../../src/ui/hooks.ts) — целое число секунд
  до разблокировки (0 = открыта); собственный `setInterval` (250 мс), останавливается
  по разблокировке.
- В `Board.tsx`: `disabled={!canEnd || locked}`, подпись с отсчётом, приглушение
  штатным `disabled`-стилем.

Константа: `END_TURN_LOCK_SECONDS = 5`.

### Гард 2 — подтверждение пустого хода

Если за ход не открыто ни одного слова, клик по «Завершить ход» открывает диалог
подтверждения вместо немедленного завершения.

- Условие: `view.clue && view.guessesRemaining === view.clue.count + 1`. Каждое верное
  открытие уменьшает `guessesRemaining`; неверное само завершает ход, поэтому до кнопки
  дело не дойдёт — значит равенство означает «открытий не было».
- `AlertDialog` (третий, рядом с «Выйти?» / «Новая игра?»): заголовок
  «Завершить ход без догадок?», текст «Вы не открыли ни одного слова. Точно завершить
  ход?», кнопки «Отмена» / «Завершить ход».
- Если открытия были — `endTurn()` вызывается сразу, без диалога.

### Взаимодействие

Пока идёт отсчёт — кнопка недоступна, тапнуть нельзя. После разблокировки клик при
пустом ходе ведёт в диалог, иначе завершает ход напрямую.

## Затрагиваемые файлы

- `src/ui/turnTimer.ts` — функция `endTurnLockLeft`.
- `src/ui/turnTimer.test.ts` — тест функции.
- `src/ui/hooks.ts` — хук `useEndTurnLock` + константа.
- `src/ui/screens/Board.tsx` — состояние диалога, обработчик клика, кнопка, `AlertDialog`.

## Тестирование

- Юнит-тест `endTurnLockLeft`: монотонное убывание, привязка к старту (не к маунту),
  не уходит ниже нуля.
- UI (блокировка, отсчёт, диалог) проверяется вручную — компонентных тестов в проекте нет.
